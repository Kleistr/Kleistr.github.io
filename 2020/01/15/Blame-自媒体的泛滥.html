Blame 自媒体的泛滥

2020-01-15 kleist
  • 吐槽2

刚刚在 Chrome 里看到一篇文章《2020 了 UI 还是半成品？Windows 10 该向国产系统学什么》，心想什么时候有了国产操作系统了！这么大的事情我这个自诩比较关心科技界新闻的咋没听过！结果打开一看赫然写着 Deepin。当然我并不是吐槽 Deepin，相反我家里的旧电脑一直都是用的 Deepin，我觉得在 Linux 的发行版中本地化做的算是不错的了，毕竟我妈都能用它刷剧，可是这篇文章让我有些不知所措了。

我不知道把 Deepin 定义为操作系统的笔者能够写出什么样的文章，看完才知道通篇在踩一捧一的论调中把两种设计风格做美观层面的主观评断，微软一直有着自己的美学，Apple 也是，Google 也是，你可以认为 Deepin 的毛玻璃效果好看，圆角且有投影效果的窗体美观，有更统一的设计理念，更贴心的拟物虚拟键盘？（拟物风格在扁平加毛玻璃的设计中是有多突兀）、更方便获取 IP 的右键菜单？（win 用户显然和 Linux 用户群体不同的，别说获取 IP 了，可能很多用户连 IP 是什么都不清楚）、自动隐藏的任务栏？（win 也有啊）、无密码登录？（蛤？？？）还吐槽了应用商店速度（Linux 没有用户验证，自然可以直接获取，win 是类似 Mac App Store 的，有验证环节，而或许安全原因或许架设 CDN 的不便导致速度慢，这或许也是当年 Xcode Ghost 的罪魁）...... 各种只能看到笔者有多喜欢 Mac 的设计（比如 “酷似 macOS 的图形阴影，很漂亮！”）

既然如此，你认为重新整合编排一套新的右键菜单或者操作模式现实么？增加用户学习成本现实么？win10 即使有了新的控制中心（sorry，太长时间不用记不清叫什么了）也依然保留有原来的控制面板。美观方面来讲，我的确对微软没感觉，我很喜欢 macOS，也一直在用 rmbp，但并不代表不可以接受微软的设计啊。

这种踩一捧一的论调还是算了吧，主观感受而已，且重新定义 “系统” 的含义也是让我震惊的了。当然，最让我震惊的是竟然有 1w+ 点阅……不过看到大家的反应我放心了
