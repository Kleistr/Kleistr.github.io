当遇到Coding发呆

2018-09-30 最后编辑 a year ago kleist
  • 技术14

之前无数次想过转移到Gayithub上，但搬迁几天就想搬回国内了……原因都了解，但这次确实让我下定决心转移了。

最近项目不是很多，故有些时间拿来看书了（PS：终于有时间看书了…），于是更新了N久没有更新过的「2018的粮食」，麻烦随之而来……

先是由于换过代理，导致git的认证出现了问题，拿ssh-keygen -l -f ~/.ssh/known_hosts瞬间解决问题；然后提交还是失败…难不成网络出现了问题？换手机热点还是不行，最后发现原来 coding 网页修改也提交不上，才想起貌似一年前出现过这种情况，很明显是 coding 的问题了……然后此前一直都有注意但是一直没在乎的icon也经常变成 coding 家的图标。

反正种种吧，让我开始搞Github Pages，倒也简单：

  1. 由于之前有作为Pages的“仓库”，省了一步
  2. 在config文件中修改deploy的值
  3. 在域名解析中添加CNAME解析 CNAME blog *.github.io
  4. 在GitHub setting中修改自定义域名
  5. 在source文件夹中新建CNAME文件，内容为自定义的域名地址
